Design
======

Assumptions: business requirements for the app are given

Objectives:

#. guide the application developer to decide on a high-level system topology in terms of

   #. domains
   #. participant nodes
   #. application backends
   #. application apis (Daml + read access)
   #. application front ends
   #. ledger service dependencies
   #. off-ledger dependencies
   #. end-user consumers
   #. service consumers
   #. IAM systems
   #. off-ledger systems to integrate with
   #. who operates what system
   #. who controls choice implementation for daml models
   #. geographical distribution of participant nodes
#. help the app dev validate that

   #. app won’t hit scalability and throughput boundaries
   #. app satisfies trust assumptions
